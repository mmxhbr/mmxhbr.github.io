Hexo + GitHub Pages 配置与发布全攻略（新手零踩坑）

对于新手而言，搭建个人博客最省心、零成本且稳定的方案，莫过于 Hexo（静态博客生成器）+ GitHub Pages（免费托管）。Hexo负责快速生成美观的静态博客页面，GitHub Pages提供免费托管和可访问域名，无需购买服务器，新手也能轻松上手。

本文将从「环境准备→Hexo初始化→主题配置→分类配置→GitHub托管→发布部署→问题排查」，手把手带你完成全流程操作，全程零冗余、零踩坑，同时重点优化博客分类设置和主题配置，让你的博客结构更清晰、外观更美观，适配新手的实际操作需求。

一、前置准备（必做，奠定基础）

在开始配置前，需先安装3个核心工具，所有工具均免费，安装步骤简单，按提示操作即可完成。

1.1 安装Node.js（Hexo运行依赖）

Hexo基于Node.js开发，必须安装Node.js才能运行Hexo相关命令，优先选择稳定的LTS版本，避免兼容性问题。

  1. 访问Node.js官网：https://nodejs.org/zh-cn/，下载「长期支持版（LTS）」；

  2. 双击安装，全程默认下一步即可（无需修改安装路径）；

  3. 验证安装成功：打开终端（Windows用CMD/PowerShell，Mac用终端），输入命令 node \-v 和 npm \-v，若能正常显示版本号（如v18.17.0、9.6.7），即为安装成功。

1.2 安装Git（GitHub提交依赖）

Git用于将Hexo生成的静态文件提交到GitHub Pages仓库，实现博客发布，安装时需注意勾选关键选项。

  1. 访问Git官网：https://git-scm.com/downloads，根据自身系统（Windows/Mac）下载对应版本；

  2. Windows安装：务必勾选「Add Git to PATH」（关键选项，方便终端直接使用Git命令），其余步骤默认下一步；

  3. Mac安装：直接双击安装，或通过Homebrew执行命令 brew install git 安装；

  4. 验证安装成功：终端输入 git \-\-version，能显示版本号即完成安装。

1.3 注册GitHub账号（博客托管用）

GitHub Pages是GitHub提供的免费静态网站托管服务，需先注册GitHub账号才能使用，步骤简单无门槛。

  1. 访问GitHub官网：https://github.com/，点击「Sign up」开始注册；

  2. 填写用户名（将作为博客域名的一部分，如xxx.github.io）、邮箱和密码，完成人机验证后即可注册成功；

  3. 注册完成后登录GitHub，后续将用于创建博客托管仓库。

二、Hexo初始化（搭建本地博客）

环境准备完成后，开始初始化Hexo博客，这一步将在本地生成完整的博客框架，后续的文章编辑、分类设置、主题配置都将在这个框架中进行。

2.1 安装Hexo-cli（命令行工具）

Hexo-cli是Hexo的命令行工具，用于快速创建、启动、生成和部署博客，全局安装一次即可终身使用。

验证安装成功：终端输入hexo \-v，能显示Hexo版本号即说明安装完成。

2.2 初始化Hexo博客项目

  1. 终端进入你想存放博客的文件夹（如桌面），执行命令创建并初始化博客项目（blog为项目名称，可自定义）：\# 进入目标文件夹（示例：进入桌面） cd \~/Desktop \# 初始化Hexo博客，项目名称为blog hexo init blog

  2. 初始化完成后，进入博客项目目录：
    cd blog

  3. 安装Hexo依赖包（初始化后会自动生成package.json，执行以下命令安装依赖）：
    npm install

  4. 启动本地博客服务，预览初始效果：
    `# 启动本地服务，默认端口4000

hexo s`

  1. 打开浏览器，访问http://localhost:4000，能看到Hexo默认的博客页面（默认主题为landscape），即说明本地博客初始化成功。

2.3 Hexo核心目录说明（新手必懂）

初始化完成后，blog文件夹下的核心目录/文件，直接关系到后续配置和文章编辑，重点关注以下3个：

  • source/\_posts：存放所有博客文章的文件夹，所有.md格式的文章都放在这里；

  • \_config\.yml：Hexo全局配置文件，博客名称、作者、主题、部署参数等都在这里修改；

  • themes：存放博客主题的文件夹，可更换主题（新手先使用默认主题，后续可优化）。

三、Hexo主题配置（新手友好，可选优化）

Hexo默认主题（landscape）简洁实用，但新手可根据自身喜好更换主题、修改主题配置，让博客外观更具个性化。以下从「默认主题配置」「主题更换」「常见主题优化」三个维度，详细说明操作步骤，全程无复杂命令，新手可逐步尝试。

3.1 默认主题（landscape）基础配置

无需更换主题，可直接修改默认主题的基础设置，适配自身需求，主要修改主题配置文件：

  1. 找到主题配置文件：打开blog目录，进入 themes/landscape/\_config\.yml（注意区分Hexo全局配置文件\_config\.yml，避免混淆）；

  2. 核心配置修改（新手重点关注，其余保持默认即可）：

    • 博客标题/副标题：找到 title: Hexo，修改为自己的博客标题（如「我的个人博客」）；找到 subtitle: A fast, simple \&amp; powerful blog framework，修改为博客副标题（如「记录学习与成长」）；

    • 作者信息：找到 author: John Doe，修改为自己的名称；

    • 菜单配置：前文提到的分类菜单、标签菜单，均在此处设置，确保 menu 配置中包含 Categories: /categories 和 Tags: /tags；

    • 头像设置：找到 avatar: ，在后面填写头像图片路径（可放本地themes/landscape/source/images文件夹，填写路径/images/头像文件名\.jpg，或填写网络图片链接）。

  3. 修改完成后，保存文件，重启本地服务（hexo s），即可看到修改效果。

3.2 更换Hexo主题（新手推荐）

Hexo有大量免费主题可供选择，新手推荐选择简洁易配置的主题（如Next、Butterfly），操作步骤如下（以Next主题为例）：

  1. 下载主题：终端进入blog目录，执行命令下载Next主题（直接克隆到themes文件夹）：
    `# 克隆Next主题到themes文件夹

git clone https://github\.com/theme\-next/hexo\-theme\-next themes/next`

  1. 修改全局配置，启用新主题：打开Hexo全局配置文件（blog目录下\_config\.yml），找到 theme: landscape，将其修改为 theme: next；

  2. 初始化主题依赖（部分主题需要）：终端进入blog目录，执行npm install hexo\-renderer\-pug hexo\-renderer\-stylus \-\-save（Next主题必备依赖）；

  3. 重启本地服务（hexo s），即可看到新主题的效果，后续可在 themes/next/\_config\.yml 中修改主题细节（如颜色、布局）。

补充说明：新手不建议同时下载多个主题，避免配置冲突；常用免费主题推荐：Next（简洁大气）、Butterfly（美观易配置）、Matery（清新文艺），下载方式均为「克隆到themes文件夹+修改全局theme配置」。

3.3 主题配置注意事项（新手避坑）

  • 区分两个配置文件：blog/\_config\.yml（Hexo全局配置，管主题选择、部署等）和 blog/themes/主题名称/\_config\.yml（主题配置，管外观、菜单等），不要混淆修改；

  • 更换主题后，若出现页面错乱、报错，大概率是缺少主题依赖，可查看对应主题的官方文档，安装所需依赖；

  • 新手建议先熟悉默认主题，再尝试更换主题，避免因配置复杂导致博客无法正常运行。

四、重点：博客分类配置（新手必做）

分类是博客的核心功能，能让博客结构更清晰，方便读者查找相关内容。Hexo的分类配置分为「全局分类显示」和「单篇文章分类绑定」，两步即可完成，操作简单。

4.1 全局分类设置（优化分类菜单显示）

默认情况下，Hexo已支持分类功能，只需确保主题菜单中开启分类选项（以默认主题landscape为例）：

  1. 打开blog目录下的主题配置文件：themes/landscape/\_config\.yml；

  2. 找到「menu」配置项，确保包含「Categories」（分类菜单），示例如下：
    `menu:

Home: /
Archives: /archives
Categories: /categories # 分类菜单，确保这一行存在
Tags: /tags`

  1. 保存文件，重启本地服务（hexo s），刷新浏览器，即可在博客顶部看到「Categories」（分类）菜单。

4.2 单篇博客分类绑定（核心步骤）

每篇博客都需要绑定分类，绑定后会自动聚合到分类菜单中，具体操作如下：

  1. 新建一篇博客，终端执行命令（博客标题可自定义）：
    hexo new \&\#34;测试分类的博客\&\#34;

  2. 打开生成的.md文件（路径：source/\_posts/测试分类的博客\.md），在文件最顶部的Front-matter中添加「categories」字段，示例如下：
    `---

title: 测试分类的博客
date: 2026-04-02 11:00:00
categories: [Hexo, 博客搭建] # 分类，可设置单级或多级
tags: [Hexo, 分类测试]
layout: post
comments: true
toc: true
---`

  1. 分类设置说明：

    • 单级分类：如 categories: \[Hexo\]，直接绑定一个分类；

    • 多级分类：如 categories: \[Hexo, 博客搭建\]，表示「Hexo」分类下的「博客搭建」子分类；

    • 分类名称可自定义，建议与博客主题相关，多篇博客绑定同一分类，会在分类菜单中聚合显示。

  2. 保存文件，重启本地服务，点击顶部「Categories」，即可看到绑定的分类及对应博客。

七、常见问题排查（新手必看）

新手配置和发布时，容易遇到一些小问题，以下是最常见的5种问题及解决方案，帮你快速踩坑。

7.1 本地服务启动失败（hexo s报错）

报错原因：依赖包未安装完整，或端口被占用。

解决方案：终端进入blog目录，执行 npm install 重新安装依赖；若端口被占用，执行hexo s \-p 4001 更换端口，再访问 http://localhost:4001。

7.2 部署失败（hexo d报错）

报错原因：deploy配置错误、GitHub仓库名称错误、网络问题。

解决方案：检查deploy配置中的repo地址是否正确、branch是否为gh-pages、「:」后面是否有空格；确认GitHub仓库名称为「用户名.github.io」且为公开仓库；检查网络，科学上网后重新执行 hexo d。

7.3 博客发布后看不到文章

原因：未生成静态文件，或Front-matter配置错误。

解决方案：重新执行 hexo clean \&amp;\&amp; hexo g \&amp;\&amp; hexo d（一键清除缓存、生成、部署）；检查Front-matter，确保layout: post、title、date字段完整，无语法错误。

7.4 分类菜单不显示

原因：主题配置中未开启分类菜单，或无绑定分类的博客。

解决方案：检查主题配置文件（themes/landscape/_config.yml），确保menu中包含「Categories: /categories」；确保至少有一篇博客添加了categories字段，重新生成并部署。

7.5 主题更换后页面错乱

原因：缺少主题依赖，或主题配置与Hexo版本不兼容。

解决方案：查看对应主题的官方文档，安装所需依赖；若仍错乱，更换适配当前Hexo版本的主题（优先选择更新时间较近的主题）。
