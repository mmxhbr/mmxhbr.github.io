对于前端新手而言，微信小程序是极佳的实战项目载体，而轻量化记账本是兼顾实用性、低开发门槛、可快速上线的首选练手项目。本文将带你从零开始，完成一个完整可用的微信小程序记账本开发，涵盖「账号注册→环境搭建→功能开发→本地调试→审核上线」全流程，全程无复杂后端开发（基于小程序本地存储实现数据持久化），代码可直接复制复用，新手也能零踩坑完成上线。

本文档为标准Markdown格式，完美适配Hexo博客发布，关键上线步骤配套截图标注说明，可直接插入对应截图使用。

一、前置准备（开发前必做）

在开始开发前，需完成账号注册、工具安装两项核心准备，全程免费无门槛。

1.1 注册微信小程序账号

微信小程序的开发、发布必须基于微信公众平台的小程序账号，个人开发者可免费注册，步骤如下：

  1. 访问微信公众平台官网：https://mp.weixin.qq.com/，点击右上角「立即注册」
  2. 账号类型选择「小程序」（关键：不要选公众号/订阅号）
  3. 按页面提示填写邮箱、密码，完成邮箱验证
  4. 主体类型选择「个人」，填写姓名、身份证号，完成微信扫码实名认证
  5. 注册完成后，登录小程序后台，在「设置」→「基本设置」中，记录你的AppID（小程序唯一标识，开发、上线必须用到）

截图说明：标注「账号类型选择-小程序」、「邮箱填写区」、「主体类型选择-个人」三个核心位置

1.2 安装微信开发者工具

微信开发者工具是官方提供的一站式开发、调试、上传工具，免费适配Windows/Mac系统：

  1. 访问官方下载地址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
  2. 根据你的系统选择对应稳定版安装包，双击安装全程默认下一步即可
  3. 安装完成后，打开工具，用微信扫码登录，即可进入开发界面

1.3 前置知识说明

本项目基于微信小程序原生框架开发，无需后端经验，只需掌握基础的HTML/CSS/JS知识即可上手：

  • 小程序的WXML对应HTML，用于页面结构编写
  • 小程序的WXSS对应CSS，用于页面样式编写
  • 小程序的逻辑层基于JavaScript开发，语法完全兼容

二、项目初始化与结构说明

2.1 新建小程序项目

  1. 打开微信开发者工具，点击左侧「小程序」→ 「+」新建项目
  2. 填写项目配置信息：
    • 项目名称：自定义，如「个人记账本」
    • 目录：选择本地空文件夹，作为项目存放路径
    • AppID：填写前文记录的小程序AppID（关键：不要选测试号，否则无法上线）
    • 后端服务：选择「不使用云服务」（本项目用本地存储实现，无需后端）
    • 模板选择：「基础模板-不使用模板」
  3. 点击「确定」，完成项目创建，工具会自动生成小程序基础框架

截图说明：标注「AppID填写区」、「项目目录选择」、「不使用云服务」三个核心配置位置

2.2 项目核心目录结构说明

新建项目后，左侧目录树的核心文件/文件夹作用如下，新手重点关注标⭐的内容：
├── app.js ⭐ 小程序全局逻辑文件，生命周期、全局数据都在这里定义
├── app.json ⭐ 小程序全局配置文件，页面路径、导航栏、底部 tab 栏都在这里配置
├── app.wxss ⭐ 小程序全局样式文件，所有页面通用的样式写在这里
├── sitemap.json 小程序搜索配置，默认生成无需修改
├── pages/ ⭐ 存放所有页面的文件夹，每个页面对应一个子文件夹
│ ├── index/ 首页（记账面板页）
│ ├── bill/ 账单列表页
│ ├── stats/ 数据统计页
│ └── category/ 分类管理页
├── utils/ ⭐ 存放通用工具函数，如日期格式化、数据计算
└── components/ 存放自定义组件，如记账弹窗、图表组件

三、需求设计与功能规划

本项目打造一个轻量化、无广告、纯本地的个人记账本，核心功能规划如下，完全满足个人日常记账需求，且符合个人小程序审核规范：

核心模块 功能详情
快速记账 支持收入/支出类型切换、金额输入、分类选择、日期/备注填写，一键保存
分类管理 预设常用收支分类，支持自定义新增/删除分类，适配不同用户需求
账单列表 按日期分组展示历史账单，支持单条账单删除、按时间筛选
数据统计 月度收支总额统计、收支分类占比饼图展示，直观查看消费情况
数据持久化 基于小程序本地存储实现，数据保存在用户手机本地，无需服务器

五、本地调试与功能测试

开发完成后，需在开发者工具和真机上完成全功能测试，确保上线后无异常。

5.1 开发者工具内调试

点击开发者工具顶部「编译」按钮，重新编译项目，查看控制台是否有报错
逐一测试所有功能：
收支记账功能：输入金额、选择分类，点击保存，查看是否提示成功
数据存储：保存账单后，重新编译项目，查看账单列表是否正常加载
页面跳转：点击底部 tab 栏，查看页面切换是否正常
边界情况：测试空金额、0 金额、超大金额等异常输入，是否有正确的校验提示

5.2 真机调试

小程序的部分能力（如存储、性能）在开发者工具和真机上有差异，必须进行真机测试：
点击开发者工具顶部「真机调试」按钮，选择「自动真机调试」
用手机微信扫码，打开真机调试面板
在手机上逐一测试所有功能，确保样式无错乱、功能正常、数据保存无丢失

六、小程序上线全流程（关键步骤 + 截图说明）

测试完成后，即可开始上线流程，全程在微信开发者工具和微信公众平台操作，分为 4 个核心步骤。

6.1 步骤 1：完善小程序基础信息（公众平台）

  1. 登录微信公众平台小程序后台，进入「设置」→「基本设置」
  2. 完善小程序信息：
  • 小程序名称：2-10 个字符，不能包含敏感词，如「极简记账本」
  • 小程序头像：尺寸 144*144px，清晰无二维码、无广告
  • 小程序介绍：简要说明功能，如「个人轻量化记账工具，支持收支记账、账单统计、分类管理」
  • 服务类目：选择「工具 - 备忘录 / 记账」（个人开发者可直接选择，无需资质）
  1. 提交信息审核，等待平台审核通过（一般 1-2 小时）

    截图说明：标注「小程序名称、头像、介绍、服务类目」四个核心填写区域

6.2 步骤 2：上传代码到公众平台（开发者工具）

  1. 回到微信开发者工具，确认项目无报错、功能测试正常
  2. 点击开发者工具右上角「上传」按钮
  3. 填写版本信息：
  • 版本号：按语义化填写，如1.0.0
  • 项目备注：填写本次上线的功能说明，如「初始版本，实现收支记账、账单列表、数据统计功能」
  1. 点击「上传」，确认后即可将代码上传到小程序后台

    截图说明：标注「版本号填写区」、「项目备注填写区」、「上传按钮」

6.3 步骤 3：提交代码审核（公众平台）

  1. 回到微信公众平台小程序后台，进入「管理」→「版本管理」
  2. 找到「开发版本」模块，即可看到刚才上传的代码
  3. 点击「提交审核」，进入审核信息填写页面
  4. 填写审核信息：
  • 功能页面：选择所有页面，填写每个页面的功能说明，如「首页：收支记账功能」
  • 测试账号：无需填写（本项目无登录功能）
  • 审核备注：简要说明小程序功能，帮助审核人员快速了解
  1. 确认信息无误后，点击「提交审核」，等待平台审核（个人小程序一般 1-3 个工作日）

    截图说明：标注「开发版本」区域、「提交审核」按钮位置
    6.4 步骤 4：发布上线

  2. 审核通过后，微信会发送服务通知提醒你审核结果
  3. 登录微信公众平台小程序后台，进入「管理」→「版本管理」
  4. 找到「审核通过，待发布」的版本，点击「发布」
  5. 选择「全量发布」，确认后即可完成上线
  6. 发布完成后，等待 10-30 分钟，即可在微信中搜索到你的小程序，所有人都可访问使用

    截图说明：标注「审核通过待发布版本」、「全量发布」按钮位置

七、常见问题与避坑指南

7.1 开发阶段常见问题

  1. 页面无法跳转 / 找不到页面：检查app.json的pages数组是否注册了对应页面，路径是否完全匹配
  2. 本地存储数据丢失：检查是否调用了wx.setStorageSync保存数据，小程序卸载后本地存储会清空，属于正常现象
  3. 样式在真机上错乱：避免使用固定 px 单位，统一使用 rpx 自适应单位，检查是否有兼容问题的 CSS 属性

7.2 审核被拒常见原因与解决方案

  1. 服务类目选择错误：必须选择「工具 - 记账 / 备忘录」类目，不要选择金融相关类目（个人开发者无资质）
  2. 功能描述不符：小程序介绍和实际功能必须一致，不要宣传未实现的功能
  3. 包含敏感内容 / 广告：个人小程序禁止添加广告、引流、支付相关功能，纯工具类功能可正常过审
  4. 缺少用户协议 / 隐私政策：若小程序收集用户数据，需在设置页添加用户协议和隐私政策，本项目纯本地存储无数据收集，无需添加
